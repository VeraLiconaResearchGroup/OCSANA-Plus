Before You Start

        Information presented here are mainly intended for algorithms developers who are willing to port their algorithm to AlgoRun for others to try. AlgoRun wraps your algorithm dependencies along with all your source code in a single container. Then, you can run your algorithm as a callable web service.


To start, you will need to download Docker on your machine. This doesn't assume any prior use of Docker, and you won't need to understand Docker in details to use AlgoRun .. To install Docker on your machine, please navigate to this link and select your machine (Mac OS X, Linux or Windows) and follow the steps. You can make sure that the Docker is installed correctly by running: docker run hello-world in the terminal ..

STEP 1

Prepare Your Algorithm


Prepare your algorithm to work in the following scenario:

  • Accept the input as a file and produce the output to another file.
  • Have a single entry point. That is, it starts execution from one point. If your algorithm start executing on multiple files, you should combine them such that the command (below) operates on one file only.
  • Your code should read the first command line argument as the file name as following:
    $ command input_file (this is executed in the terminal to run your algorithm)
commandThe command you use through terminal to run your algorithm
input_fileThe input file name your algorithm read and process

Wrap Your Algorithm in a Container


We have provided a ready-made wrapper (a package) for you to put your algorithm in directly. Download the file in this link and uncompress it ..
When you open the directory, you should be able to figure out the folders as following:

  • Put all your algorithm source code in src folder. Include any libraries your main file calls
  • Then, edit the Dockerfile. Open the file and follow the instructions in it. In this file, you are going to install algorithm dependencies.
    Example: RUN apt-get install -y wget ruby2.0 in case your algorithm is written in Ruby language and needs to download some packages using wget ..
    Algorithm dependencies are any packages that your source code requires to run correctly ..
  • Finally -and most important-, tell AlgoRun how to it should execute your code by editing the manifest in the algorun_info folder. Below is an illustration of the manifest fields.
manifest.json
algo_name the name of your algorithm
algo_summary a 1-line summary of your algorithm
algo_description a few lines description of your algorithm (basic html accepted)
algo_website url where users can find more information about your algorithm
algo_keywords a list of keywords describing your algorithm
algo_authors a list of all the algorithm's authors
algo_exec the command to start algorithm execution. AlgoRun passes the input file as the first argument
algo_output_filename path of the file where your algorithm outputs its results
algo_parameters ? list all the parameters available for your algorithm. These parameters will be available for use in your algorithm as environment variables.
algo_image Example: "algorun/react". leave "" if you did not upload your algorithm to docker hub or if you do not know what docker hub is
input_example.txt
Fill it with the format of the input your algorithm accepts. This serves as a guidance users to know how to input data
output_example.txt
Fill it with the format of the output your algorithm produces. This gives the user a glance on what result he will get

STEP 2

What are algorithm parameters (algo_parameters)?

[OPTIONAL] In some cases, your algorithm might use some configuration. This configuration should be presented as parameters in a key-value pair (parameter name, parameter value). To provide default configuration, we make use of environment variables to save the default values of these parameters. Your algorithm then reads default configuration from environment variables. In the provided web interface, the user can easily change these parameters and run the algorithm again with the new configuration.


STEP 3

Build and Run Your Container


  • In your working directory (where the Dockerfile exists), use the following command to build your container:
    docker build -t <username>/<algorithm> .
    Where <username> is your Dockerhub username. If you don't have one yet, you can use your algorithm name only ..
  • Make sure that you see the build successfull message ..
  • Use the following command to run your container:
    docker run -it -p 31331:8765 <username>/<algorithm>
  • Open your web browser and go to http://localhost:31331 to see the automated web interface of your algorithm and try it ..
